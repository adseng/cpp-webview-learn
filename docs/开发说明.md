# 开发说明

## 目录说明

- `app/frontend`：Vite + React + Tailwind 前端工程。
- `app/backend`：C++17 + webview 宿主工程（CMake）。
- `docs/通信协议.md`：前后端 JSON 协议定义。

## 开发模式（使用前端 dev server）

1) 安装前端依赖

```powershell
pnpm install
```

2) 启动前端开发服务器

```powershell
pnpm --dir app/frontend dev
```

3) 构建 backend 宿主程序（首次或代码更新后）

```powershell
cmake -S app/backend -B app/backend/build
cmake --build app/backend/build --config Release
```

4) 启动桌面宿主并加载 dev server

```powershell
$env:WEBVIEW_DEV_SERVER_URL = "http://127.0.0.1:5173"
.\app\backend\build\Release\cpp_webview_host.exe
```

## 生产模式（后端内置本地静态 HTTP 服务）

1) 构建前端静态资源

```powershell
pnpm --dir app/frontend build
```

2) 关闭 dev server 环境变量并运行宿主

```powershell
Remove-Item Env:WEBVIEW_DEV_SERVER_URL -ErrorAction SilentlyContinue
.\app\backend\build\Release\cpp_webview_host.exe
```

backend 会自动：
- 从 `app/frontend/dist` 提供静态资源服务；
- 使用 `127.0.0.1:0` 动态分配可用端口；
- 导航到 `http://127.0.0.1:<动态端口>/index.html`。

3) 可选：固定端口优先（便于调试）

```powershell
$env:WEBVIEW_STATIC_PORT = "18080"
Remove-Item Env:WEBVIEW_DEV_SERVER_URL -ErrorAction SilentlyContinue
.\app\backend\build\Release\cpp_webview_host.exe
```

如果 `WEBVIEW_STATIC_PORT` 被占用，宿主会自动回退到动态端口。
